proc 利用

ctf中 proc的利用

proc 目录是linux上的一个文件夹 它相当于一个linux的一个虚拟的一个文件系统 它是以进程号为分割单位的目录系统

为什么说它是以进程号 (PID) 为分割单位的文件系统呢

看到这里面的目录名字就是进程号的名字

现在我查看一个进程的文件系统

这里总结网上几个常见的 还有ctf web题中用到比较多的 进程文件

cmdline

cmdline文件里面保存着 启动当前进程的完整命令

cwd

一个相当于一个软链接吧 连接到当前进程启动文件的目录

但是感觉这个并没有太大的作用

exe

也是个软链接 指向 启动该进程的可执行文件 这里我是用python 起的 指向 python

environ

查看当前进程的环境变量

通关这种方式可以查看当前进程的env 有些把flag写入环境变量的题目 可以直接非预期

fd

fd 是一个目录 里面也都是通关该进程打开的文件

当然我这个进程里面没有打开其他的文件 并没有其他的文件

这个里面有个考点就是 如果一个程序 打开了某个文件 然后删除了文件 但是并没有去 关闭这个文件 在 我们的fd目录里面还是有他的文件的连接的

self

/proc/self 就是代表着当前进程的目录 比如 我刚才 利用的 python 1.py 启动了一个进程 然后 如果有一个文件读取漏洞的话 我就可以利用 这个proc/self来查询我当前这个（python 1.py）启动的进程的pid 就是不需要我们说动的去寻找了这个进程了

然后可以配合我们上面的 几个目录 来快速查找一些信息
